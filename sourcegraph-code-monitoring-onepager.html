Code Monitoring: The Missing Piece of Your Sourcegraph Workflow

1 From Reactive Searching to Proactive Monitoring

Code Monitoring transforms your existing Sourcegraph searches into intelligent, automated watchers that continuously scan your entire codebase. Instead of manually hunting for issues after they're introduced, it proactively alerts you the instant problematic patterns, security vulnerabilities, or policy violations are committed across any repository.

2 Perfect Use Cases for Your Team

Security & Compliance

  • Monitor for hardcoded secrets
  • Track deprecated API usage
  • Watch for compliance violations
  • Detect insecure patterns

Code Quality & Standards

  • Catch anti-patterns early
  • Monitor coding standard violations
  • Track technical debt introduction
  • Watch critical dependency changes

Example: Security Monitoring

Search: password\s*=\s*["'][^"']{8,}["']
Alert: Get notified instantly when hardcoded passwords are committed, before they reach production.

Example: Architecture Governance

Search: import.*internal.*from.*external
Alert: Catch architecture boundary violations as they happen, maintaining clean separation of concerns.

3 Code Monitor Triggers & Setup

Code Monitors in Sourcegraph run searches based on the trigger configuration you set up. There are two main types of triggers:

For scheduled triggers, you can specify how often the search should run:

  • Every 5 minutes
  • Hourly
  • Daily
  • Weekly

For event-based triggers, the search runs in response to specific events:

  • When new commits are pushed to a repository
  • When new changesets (pull/merge requests) are created
  • When changesets are updated

Setting Up a Code Monitor

To create a code monitor with a specific frequency:

  • Navigate to the Code Monitoring page in Sourcegraph
  • Click "Create code monitor"
  • Define your search query
  • Select your trigger type and frequency
  • Configure your desired actions (email, webhook, Slack notification)
